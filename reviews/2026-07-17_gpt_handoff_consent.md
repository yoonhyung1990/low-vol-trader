# v0.4 핸드오프 — GPT 검수 (조건부 합의)

- 일자: 2026-07-17
- 대상: LOW-VOL-CONSTITUTION-V0.4-CLOSEOUT 핸드오프 (실행 게이트 검수)

판정: 조건부 합의 — 아직 Claude Code에 실행시키면 안 됨

핸드오프의 GPT v0.3 반영 항목, 브랜치·PR 구조, 커밋 분리, hook 수정은 모두 동의한다. C-g 방향도 맞다. 다만 현재 파일에는 우리가 방금 합의한 핵심 원칙 하나가 실제 WORKFLOW 작업 항목에서 빠졌고, Recovery Canary에 반복 손실 우회가 남아 있다.

1. "실행 전 양측 합의"가 WORKFLOW v1.1 작업에 빠짐

Claude는 대화에서 다음을 문서화하겠다고 했다.

큐·핸드오프는 실행 전에 GPT·Claude 합의와 yh 승인을 받아야 한다.

그런데 실제 작업 12에는 PR 병합 전 검수만 있고, Claude Code 실행 전 합의 규칙은 없다. 사용자가 지적한 바로 그 절차가 다시 누락됐다.

WORKFLOW에 두 게이트를 분리해 넣어야 한다.

실행 게이트: 핸드오프 내용을 GPT·Claude가 합의하고 yh가 실행 승인
병합 게이트: Claude Code 실행 결과의 실제 PR diff를 GPT·Claude가 검수하고 yh가 merge 승인

사전 설계 합의와 사후 구현 검수를 같은 것으로 취급하면 안 된다.

2. C-g는 맞지만 "반복 충전" 우회가 남음

현재 C-g는 Recovery Canary 자본에 손실 한도를 두지만, 후보를 폐기한 뒤 새 후보에 다시 10%를 배정하는 행위를 막지 않는다.

예를 들면:

후보 A 카나리 손실 → 폐기
후보 B에 다시 10% 배정 → 손실
후보 C에 다시 10% 배정 → 손실

새 Risk Epoch가 시작되지 않은 상태에서 이것이 반복되면 HWM 방어 없이 계좌 손실이 계속 누적될 수 있다.

따라서 다음 원칙도 커널에 필요하다.

Recovery Canary 승인 시 Recovery Validation Window 개시
시작 시점의 계좌 순자산과 총 회복 손실 예산을 고정
카나리 배정 상한은 총자산 10%
손실 후 자본 재충전·증액 금지
실현·미실현 손실과 거래비용을 모두 누적
후보가 바뀌어도 같은 Window의 누적 손실은 초기화하지 않음
누적 손실 한도 도달 시 모든 Recovery Canary 중단
이후 재시도는 별도 yh 승인과 새 검토 없이는 금지
구체 수치는 SPEC-RISK에 위임

새 정식 Risk Epoch는 아니지만, 회복 단계 자체에도 독립적인 누적 손실 장부가 있어야 한다.

3. Claude 재검 문서의 승인 문구 정밀화

현재 부록 B 마지막은:

yh가 본 재검이 반영된 큐의 실행을 승인함으로써…

라고 되어 있어 조건부 문장이긴 하지만, 기록상 더 명확하게 만드는 편이 좋다.

yh가 최종 합의된 본 핸드오프를 Claude Code에 전달하는 행위를 실행 승인으로 간주한다. PR 병합 승인은 실행 승인과 별개이며, 실제 diff에 대한 GPT·Claude 검수 후 yh가 직접 수행한다.

이렇게 해야 실행 승인과 merge 승인이 섞이지 않는다.

Claude에게 아래만 보내서 핸드오프를 갱신하면 된다.

GPT 검수 결과: 현재 핸드오프는 조건부 합의다. 아래 3건만 수정하고 다시 파일로 줘. 아직 Claude Code에는 실행시키지 않는다.

WORKFLOW v1.1 작업 항목에 실행 게이트와 병합 게이트를 분리해 명시:
큐·핸드오프는 Claude Code 실행 전에 GPT·Claude 양측 합의와 yh 실행 승인을 받아야 한다.
Claude Code 실행 후에는 실제 PR diff를 GPT·Claude 양측이 검수하고, yh가 별도로 merge 승인한다.
사전 핸드오프 합의는 사후 PR diff 검수를 대체하지 않는다.
이 규칙은 본 public governance 저장소의 CONSTITUTION.md, WORKFLOW.md, reviews 및 수동 서명 governance 문서 변경에 적용한다.
C-g에 반복 충전 방지 원칙 추가:
Recovery Canary 승인 시 Recovery Validation Window를 개시한다.
시작 계좌 순자산과 회복 단계 누적 손실 예산을 고정한다.
총자산 최대 10% 배정 상한과 별도로, 손실 후 자본 재충전·증액을 금지한다.
실현·미실현 손실과 실행비용을 누적하며, 후보 교체 시에도 누적값을 초기화하지 않는다.
누적 손실 한도 위반 시 모든 Recovery Canary를 중단하고 자본보존 상태로 복귀한다.
이후 재시도는 yh의 별도 승인과 재검토 전에는 금지한다.
구체 수치·산식은 SPEC-RISK에 위임한다.
부록 B 마지막 승인 문구를 아래로 교체:
"yh가 최종 합의된 본 핸드오프를 Claude Code에 전달하는 행위를 실행 승인으로 간주한다. PR 병합 승인은 실행 승인과 별개이며, 실제 diff에 대한 GPT·Claude 검수 후 yh가 직접 수행한다."

나머지 GPT v0.3 반영 항목, C-g의 기본 방향, 브랜치·PR 방식, 커밋 3건 분리, hook 개선은 전부 합의한다.

이 세 항목이 반영된 수정본이면 실행 전 양측 합의 요건까지 닫힌 최종 핸드오프가 된다.
