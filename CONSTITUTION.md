# 저변동 자동매매 운용 헌법

- 버전: v0.3
- 작성일: 2026-07-17
- 상태: GPT 재검수 대기

## 문서 규칙

- 이 문서가 단일 진실 원천이다. 설계 논의는 대화로 하되, 확정은 이 문서의 diff로만 반영한다.
- 검수·수정 요청은 항목 ID(예: 1-3, D-2)로 지칭한다.
- 버전은 변경 시마다 v0.x로 올리고 변경 이력에 사유를 남긴다.
- 이 문서에는 규칙만 담는다. 통계 항목 목록, DB 스키마, 폴더 구조 같은 구현 상세는 SPEC-* 별도 문서로 분리한다.
- 편집 규칙: 조항은 원칙을 1~3문장으로 기술하고, 수치·절차·산식은 해당 SPEC-* 문서명을 지정해 위임한다.
- 커널(1장) 한도를 집행하는 SPEC 파일은 수동 서명 대상이며, 자동 프로세스가 수정할 수 없다.
- 충돌 시 우선순위: 1장 커널 > 본 문서 기타 조항 > SPEC 문서 > 코드.
- 표기: 【yh확정】= 발주자 확정, 변경 불가. 【D-x】= 발주자 결정 대기. 【S-x】= 외부 실사 필요. 표기 없는 항목은 전부 '제안' 상태이며 교차 검수 대상.

## 0장. 프로젝트 정의

- 0-1. 목적: 【yh확정】 1차 목적은 세후 KRW 기준 자본 보존과 벤치마크 초과수익이며, 자본 보존을 수익보다 상위 제약으로 둔다. 자동매매 실험실은 이를 달성하기 위한 수단이다. 합격선: (a) 최대 낙폭이 커널 한도 미만, (b) 세후·비용 차감 성과가 Benchmark A(0-3)보다 높을 것, (c) Benchmark B(0-3) 대비 위험조정 성과가 나쁘지 않을 것, (d) 조건 미충족 시 무거래 상태 유지.
- 0-2. 기준통화와 손익 계층: 기준통화는 KRW. 손익은 4계층으로 동시 기록한다 — ①총수익(세금·비용 전) ②실행순수익(수수료·스프레드·슬리피지 차감) ③추정 세후 수익(현재 세법 모델 적용) ④실현 세후 수익(실제 원천징수·납부 반영). 1장 리스크 계산(낙폭 단계·HWM)에는 브로커 계좌의 실제 평가 순자산(현금 + 평가 포지션 − 확정 수수료 − 실제 원천징수 세액)을 사용하며, 아직 매도하지 않아 발생하지 않은 추정 세금은 HWM 낙폭 계산에 넣지 않고 공식 성과 보고서에만 별도 반영한다. 거래 실행 판정(3-5)은 3층(추정 증분세금 차감) 기준이며, 4층은 사후 보고·보정 전용이다. 계층별 산식은 SPEC-TAX에 위임한다.
- 0-3. 공식 벤치마크(이중): Benchmark A — 실제 투자 가능한 세후 현금성 상품. Benchmark B — 동일 자산 유니버스의 고정 비중 정적 포트폴리오. 이 시스템의 성과는 두 벤치마크 대비로 평가한다. 상품은 S-3에서 확정하되, 백테스트 기간 전체에 생존한 현재 상품을 과거에도 존재한 것처럼 사용하는 생존편향은 금지한다.
- 0-4. 거래 계좌: v1은 국내 인가 증권사 1개 계좌만 사용한다. 1순위 한국투자증권 KIS Developers 【S-1】. 외국 브로커(IBKR 등)는 법률·외국환·세무에 대한 서면 검토 전까지 프로젝트 정책상 금지한다. 미수 방지는 이중 방어로 한다 — 1차: 계좌 자체를 증거금률 100%로 설정(가능 여부 S-1), 2차: 프로그램 차원의 주문 전 현금 가능 금액 검증과 미수 발생 가능 주문조건 차단(1-22).
- 0-5. 자동학습 원칙: 【yh확정】 일일 통계를 전량 수집하고, 주기적 학습 결과를 검증 파이프라인 통과 시에만 자동 반영한다. 학습이 커널(1장)을 수정하는 것은 금지.

## 1장. 불변 리스크 커널

이 장의 어떤 항목도 자동 프로세스(학습·승격·설정 갱신 포함)가 수정할 수 없다. 수정은 yh의 수동 결정 + 버전업 + 변경 이력 기재로만 가능하다. 이 장의 한도를 집행하는 SPEC 파일도 수동 서명 대상이다(문서 규칙 참조).

### 절대 금지
- 1-1. 직접 선물·옵션·CFD·마진 계약과 레버리지·인버스 ETF/ETN은 금지한다. 순자산 대비 목표 기초자산 노출이 1배 이하인 비레버리지 선물추종 ETF는 유니버스 실사(S-3)와 yh 승인을 거친 경우에만 허용한다. "1배"는 상품 이름이 아니라 실제 내재 총노출 기준으로 검사한다.
- 1-2. 공매도 금지.
- 1-3. 미실현 손실이 커졌다는 사실만을 이유로 목표 비중을 확대하는 규칙(물타기·마틴게일)은 금지한다. 모든 비중 확대는 독립적으로 계산된 목표 배분, 신호, 상관관계 및 리스크 게이트에 의해서만 가능하다. 즉 손실 기반 물타기는 금지하되, 모델 기반 리밸런싱은 허용한다.
- 1-4. 출금 API 사용 금지. 주문·조회 권한만 허용.
- 1-5. LLM의 직접 주문 결정 금지. LLM은 리포트·설명·이상 징후 요약만 담당.

### 노출 한도
- 1-6. 최소 현금성 비중 30%. 위험자산 총노출 최대 70%.
- 1-7. 단일 자산 최대 30%. 통화 노출은 둘로 분리해 집행한다 — (a) 직접 통화 슬리브 상한 20%, (b) 전체 포트폴리오의 통화 요인 총노출 상한(수치는 【D-5】, D-3와 동시 결정). 요인 분해 산식은 SPEC-RISK에 위임하며, 산식 확정 전에는 언헤지 상품의 통화 노출을 100% 귀속하는 보수적 근사로 집행한다.
- 1-8. 신규 승격 모델(카나리)에 배정하는 자본은 총자산의 최대 10%.
- 1-9. AI 예상수익이 최종 점수에서 차지하는 비중 상한 10% (충분한 실증 후 yh 승인 시 최대 20%, 그 이상 불가).

### 기간별 손실 한도
- 1-10. 일간 순손실 -0.75%(전 거래일 공식 종가 순자산 대비, 평가손익 포함·입출금 조정) 도달 시 당일 신규 거래 중단.
- 1-11. 주간 순손실 -2.0%(전주 마지막 거래일 종가 순자산 대비, 평가손익 포함·입출금 조정) 도달 시 다음 주 위험예산 50% 축소. 위험예산 복구는 회복 조건과 냉각기간을 충족해야 한다(세부는 SPEC-RISK).
- 1-12. 월간 순손실 -4.0%(전월 마지막 거래일 종가 순자산 대비, 평가손익 포함·입출금 조정) 도달 시 현행 모델 동결, 챌린저 재검증. 동결 해제·위험예산 복구 조건은 SPEC-RISK에 위임한다.
- 1-13. 단일 자산 낙폭 -4.0%p 도달 시 해당 자산 신규 확대 금지. 판정 기준은 자산 가격 자체의 낙폭이 아니라 해당 슬리브의 계좌 기여 손익(실현+미실현, Risk Epoch 내 누적)이다. 단일 전략 낙폭 -5.0% 시 전략 폐기 규칙은 전략별 가상 슬리브 장부가 구현된 뒤 적용하며, 그 전에는 SPEC-RISK 예정 항목으로 둔다.

### 낙폭 단계 방어
- 1-14. 낙폭 기준: 입출금 조정 후 순자산(0-2의 리스크 NAV)의 운영 High-Water Mark(KRW 기준) 대비. 운영 HWM은 Risk Epoch(1-23) 단위로 관리한다. 【yh확정】 HWM 기준은 누적 수익 상태에서도 발동할 수 있음을 인지하고 승인함(D-2, Risk Epoch 신설 조건부).
- 1-15. 단계별 조치는 명시적 상태 머신(NORMAL / CAUTION / DEFENSIVE / CAPITAL_PRESERVATION / GENERATION_RETIRED)으로 집행한다. 상태 매핑·복구 히스테리시스 수치는 SPEC-RISK에 위임한다.
  - -2%: 신규 비중 확대 크기 75%로 축소
  - -4%: 신규 비중 확대 크기 50%로 축소
  - -6%: 위험 노출 상한 25%로 축소, 현행 모델 동결
  - -8%: 신규 거래 전면 중단, 위험자산 축소 개시
  - -10%: 【yh확정】 목표 위험노출을 0으로 설정하고 가격 보호 규칙(1-22) 아래 정리 주문을 계속 수행. 미체결 전부 취소, 현행 세대(Champion+Challenger 전부) 영구 폐기, 자본보존 상태 전환. 미체결·거래정지·가격제한으로 즉시 정리가 불가능한 잔량은 해소될 때까지 신규 위험 거래 금지.
  - 축소 비율(75%·50%)은 양의 목표 비중 변화량에만 곱한다. 매도·위험축소 주문은 이 비율로 축소하지 않는다.
  - 히스테리시스: 단계가 한 번 내려가면 낙폭이 한 단계 이상 회복되고 일정 거래일 유지되기 전 자동 복귀하지 않는다. -10%는 복귀 불가이며 새 Risk Epoch(1-23)가 필요하다.
  - 단계 발동 전 데이터 검증을 선행한다. 비정상 호가 한 건만으로 청산하지 않으며, 데이터 신뢰성이 의심되면 우선 신규 주문을 정지하고 유효한 계좌·가격 스냅샷으로 재확인한다.
- 1-16. -8%~-10% 구간의 갭 하락(장 시작 급락) 시나리오는 백테스트·스트레스 테스트에 의무 포함한다.
- 1-17. 재진입 게이트: -10% 발동 후에는 (a) 자본보존 상태 최소 1개월 유지 — 이는 독립 조건이 아니라 최소 냉각기간이며 4장 검증 통과 기간에 포함될 수 있다, (b) 새 세대가 4장 검증 파이프라인 전체를 처음부터 통과, (c) yh 수동 승인 1회 — 세 조건을 모두 만족해야 재진입한다. 재진입 시에는 반드시 새 Risk Epoch와 새 세대 ID를 발급한다(1-23).

### 무결성 차단
- 1-18. 데이터 누락·이상, 거래소/API 장애 감지 시 신규 주문 전면 차단. 잔고 대조는 정규화·결제일·수수료·세금·분배금 반영 후에도 설명되지 않는 잔고 또는 수량 차이가 존재하면 신규 주문을 차단한다. 미해명 차이의 허용치는 0이다. 대조 절차는 SPEC-EXEC에 위임한다.
- 1-19. 모든 주문에 고유 ID 부여, 중복 주문 자동 차단. 주문 이벤트는 append-only 기록.
- 1-20. 시간은 내부 UTC 저장, 표시만 KST 변환.

### 신설 (v0.2)
- 1-21. 보안: API 키·Secret·토큰의 저장소·로그 기록 금지, 운영 키와 모의투자 키 분리, 출금 권한 금지, 가능한 경우 IP 허용목록 적용을 원칙으로 한다. 키 유출 의심 시 주문 차단과 즉시 폐기, 비상정지 권한의 학습 프로세스 분리, 커널 파일·서명의 자동 변조를 차단하는 파일 해시 검사를 의무화한다. 세부 절차는 SPEC-SECURITY에 위임한다.
- 1-22. 주문 가격 보호: 신용·미수·대출 주문을 금지하고, 주문 전 현금 가능 금액 검증과 미수 발생 가능 주문조건 차단을 의무화한다. 시장가 주문은 기본 금지하며, 호가 스프레드·거래대금·괴리율 기준 미달 시 거래하지 않고, 주문 가격의 최대 허용 이탈폭과 부분 체결·미체결·정정 횟수 제한을 둔다. 위험축소 주문은 일반 신규매수보다 우선하며, 거래정지·VI·가격제한 상황 처리를 포함한 수치·절차는 SPEC-EXEC에 위임한다.
- 1-23. Risk Epoch: 【yh확정】 운영 HWM·낙폭 단계(1-14, 1-15)·슬리브 기여 손익(1-13)은 Risk Epoch 단위로 계산한다. -10% 발동 시 기존 Risk Epoch를 종료하고, 새 세대가 재진입 게이트(1-17)를 통과하고 yh가 승인한 시점의 순자산으로 운영 HWM을 재설정하여 새 Risk Epoch를 개시한다. 과거 HWM은 전체 성과 기록용으로 영구 보존한다.
- 1-24. 자산별 절대 상한(커널 소속): 금 30%, 미국 달러 20%, 엔 15%, 현금성 최소 30%(1-6과 동일). 신규 자산의 상한 신설·변경은 yh 수동 결정으로만 가능하다.

## 2장. 자산 유니버스 (v1)

이 장은 유니버스와 상품 연결만 담는다. 자산별 절대 상한 수치는 1장 커널(1-24) 소속이다.

- 2-1. 국내상장 상품만 사용한다. 근거: 단일 장 마감(일일 파이프라인 단순화), 직접 외화환전과 해외시장 주문·결제 복잡성 제거, 0-4 계좌 원칙과 일치. 상품 내부의 환율 노출·헤지 비용·롤 비용은 별도로 계측한다.
- 2-2. v1 슬리브 (티커는 【S-3】 실사 후 확정, 상한은 1-24):

## 3장. 운용 문법

- 3-1. 배분형 단일 문법을 사용한다. 개별 포지션 손절/익절 없음. 방어는 1장 낙폭 단계로 일원화한다.
- 3-2. 신호 계산 일 1회(일봉 기준). 정규 리밸런싱 주 1회. 위험 리밸런싱(1장 조치)은 즉시. 미래참조 금지: T일 공식 종가와 T일까지 확정된 데이터로 계산한 정규 신호는 T+1 거래일 이전에 체결된 것으로 간주할 수 없다. 기본 체결 모델은 T+1 시가 또는 사전에 정의한 시간대의 거래량가중가격이며, 동일 종가 체결 가정은 금지한다. 위험 리밸런싱은 실시간 검증 가격을 사용할 수 있으나 정규 전략 신호와 코드 경로를 분리한다.
- 3-3. 목표 비중과 현재 비중 차이 2%p 미만이면 거래하지 않는다. 개별 자산 밴드만으로는 다자산 동시 반복 거래를 막지 못하므로 주간 총회전율 상한을 둔다. 밴드 폭과 회전율 상한 수치는 백테스트에서 거래 비용·추적 오차 확인 후 확정하며(현재 제안 상태), 수치는 SPEC-EXEC에 위임한다.
- 3-4. 자산당 비중 조정 하루 최대 1회, 전체 신규 거래 하루 최대 4건 — 이 제한은 신규 위험 확대 주문에 적용하며, 손실 한도에 따른 매도·취소·정정 등 위험축소 주문은 횟수 제한에서 제외한다. 분할 주문과 반복 정정은 하나의 부모 주문 기준으로 집계해 제한 우회를 차단하며, 동일 자산 비중 재확대는 최소 2거래일 대기한다.
- 3-5. 거래 실행 조건(전부 충족): 예상 세후 순수익(= 예상 총수익 − 실행비용 − 예상 증분세금, 0-2의 3층 기준)이 0보다 크고, 예상 실행비용의 3배와 모델 불확실성 버퍼 중 큰 값을 초과할 것 / 신호 신뢰도 기준은 확률 보정 검증을 통과한 모델에만 적용하고 룰 기반 전략에는 별도 점수 임계값을 사용(수치는 SPEC-EXEC) / 데이터 무결성 정상 / 시장 국면 ≠ SHOCK/UNCERTAIN / 손실 한도 미도달. 세금은 고정 거래비용처럼 배수 공식에 곱하지 않는다.
- 3-6. 무거래 상태를 정상 판단으로 인정한다.
- 3-7. 통계용 라운드트립 정의: 특정 자산의 비중 증가 시작 시점부터 해당 증가분의 청산 완료 시점까지.
- 3-8. 비중 결정 기본: 역변동성 배분 × 신호 강도 × 국면 신뢰도, 이후 상관관계·집중도·자산 상한 반영. 신호가 음수면 공매도가 아니라 목표 비중 0으로 절단하고, 잔여 비중은 현금이 자동 흡수한다. 주간 최대 비중 변화량·총회전율 상한(3-3)과, 추정 변동성이 비정상적으로 낮아져 비중이 폭증하는 것을 방지하는 변동성 하한을 적용한다(수치는 SPEC-RISK).

## 4장. 학습·승격 거버넌스

- 4-1. 학습 데이터: 시장 전체 데이터를 주 교재로, 자기 운용 데이터(신호·거절 사유·실체결 비용·MFE/MAE)는 비용·슬리피지·체결 보정용으로 사용한다. 실행된 거래만으로 학습하지 않는다.
- 4-2. 프로덕션 Champion은 항상 1개. 신규 모델은 Challenger로 시작: TRAINED → BACKTESTED → VALIDATED(워크포워드, purge/embargo 적용) → SHADOW → CANARY → CHAMPION. 후보 난립 통제: 생성된 모든 Challenger와 실패 결과를 기록하고, 분기당 평가 가능한 후보 수에 상한을 두며(수치는 SPEC-LEARNING), 동일 데이터에 대한 무제한 재탐색을 금지한다. 현재 검증 중인 Challenger를 새 월간 모델이 자동 대체할 수 없으며, Champion 이전 버전은 즉시 롤백 가능하도록 보존한다.
- 4-3. 챌린저 비교는 거래 건수 기반 지표(PF·승률)가 아니라 **섀도/카나리 포트폴리오의 일간 수익률 시계열**로 한다. 저빈도 운용에서 거래 표본은 통계적으로 무의미하다.
- 4-4. 검증 기간: 섀도 최소 60거래일, 카나리 최소 60거래일(각 약 1분기). 기간 내 최소 2개 이상의 시장 국면을 경험해야 하며, 한 국면만 경험했다면 기간을 연장한다. 국면 판정은 검증 대상 시스템의 국면 분류기가 아니라 사전 정의된 객관 규칙(실현 변동성 분위, 벤치마크 낙폭 발생 여부 등, 기준은 SPEC-LEARNING)으로 한다. 카나리 자본은 1-8에 따라 최대 10%.

- 4-6. 스트레스 시나리오는 공개 고정셋(개발 중 진단용)과 봉인 홀드아웃(승격 전 최종 평가용)으로 분리한다. 봉인은 절차로 구현한다: 구간 정의를 해시로 고정, 승격 시점에만 자동 평가, 결과는 pass/fail만 기록, 상세 수치의 학습 파이프라인 유입 금지. 공개 고정셋(2013 금 폭락, 2020년 3월, 2022 강달러 급등 구간 등)에서 모든 챌린저는 파국적 실패(구간 낙폭이 커널 한도 초과) 없이 통과해야 한다. 구간 추가는 사유·버전 기록과 함께 가능, 삭제는 금지.
