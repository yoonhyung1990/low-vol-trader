## 문서 규칙

- 이 문서가 단일 진실 원천이다. 설계 논의는 대화로 하되, 확정은 이 문서의 diff로만 반영한다.
- 검수·수정 요청은 항목 ID(예: 1-3, D-2)로 지칭한다.
- 버전은 변경 시마다 v0.x로 올리고 변경 이력에 사유를 남긴다.
- 이 문서에는 규칙만 담는다. 통계 항목 목록, DB 스키마, 폴더 구조 같은 구현 상세는 SPEC-* 별도 문서로 분리한다.
- 충돌 시 우선순위: 1장 커널 > 본 문서 기타 조항 > SPEC 문서 > 코드.
- 표기: 【yh확정】= 발주자 확정, 변경 불가. 【D-x】= 발주자 결정 대기. 【S-x】= 외부 실사 필요. 표기 없는 항목은 전부 '제안' 상태이며 교차 검수 대상.

## 0장. 프로젝트 정의

- 0-1. 목적: 【D-1】 수익 우선 vs 자동매매 실험실(학습) 우선. 이 결정에 따라 합격선과 실거래 전환 기준이 달라진다.
- 0-2. 기준통화: 모든 수익률·변동성·낙폭·성과 지표는 **세후 KRW 기준**으로 계산한다.
- 0-3. 공식 벤치마크: KRW 파킹형 단기자금 수익률. 이 시스템의 성과는 벤치마크 대비 초과수익으로 평가한다.
- 0-4. 거래 계좌: 국내 인가 증권사 1개 계좌만 사용한다. 1순위 한국투자증권 KIS Developers 【S-1】. 해외 브로커(IBKR 등) 사용 금지 — 국내 거주자의 외화증권 매매는 국내 인가 투자중개업자 경유가 원칙(자본시장법 시행령 184조 해석 리스크).
- 0-5. 자동학습 원칙: 【yh확정】 일일 통계를 전량 수집하고, 주기적 학습 결과를 검증 파이프라인 통과 시에만 자동 반영한다. 학습이 커널(1장)을 수정하는 것은 금지.

## 1장. 불변 리스크 커널

이 장의 어떤 항목도 자동 프로세스(학습·승격·설정 갱신 포함)가 수정할 수 없다. 수정은 yh의 수동 결정 + 버전업 + 변경 이력 기재로만 가능하다.

### 절대 금지
- 1-1. 레버리지 1.0배 초과 금지. 선물·CFD·마진 금지.
- 1-2. 공매도 금지.
- 1-3. 물타기·마틴게일·손실 중 포지션 확대 금지.
- 1-4. 출금 API 사용 금지. 주문·조회 권한만 허용.
- 1-5. LLM의 직접 주문 결정 금지. LLM은 리포트·설명·이상 징후 요약만 담당.

### 노출 한도
- 1-6. 최소 현금성 비중 30%. 위험자산 총노출 최대 70%.
- 1-7. 단일 자산 최대 30%. 단일 통화 익스포저 최대 20%.
- 1-8. 신규 승격 모델(카나리)에 배정하는 자본은 총자산의 최대 10%.
- 1-9. AI 예상수익이 최종 점수에서 차지하는 비중 상한 10% (충분한 실증 후 yh 승인 시 최대 20%, 그 이상 불가).

### 기간별 손실 한도
- 1-10. 일간 순손실 -0.75% 도달 시 당일 신규 거래 중단.
- 1-11. 주간 순손실 -2.0% 도달 시 다음 주 위험예산 50% 축소.
- 1-12. 월간 순손실 -4.0% 도달 시 현행 모델 동결, 챌린저 재검증.
- 1-13. 단일 전략 낙폭 -5.0% 시 해당 전략 폐기. 단일 자산 낙폭 -4.0% 시 해당 자산 신규 확대 금지.

### 낙폭 단계 방어
- 1-14. 낙폭 기준: 입출금 조정 후 순자산의 최고점(High-Water Mark, KRW 기준) 대비. 【D-2】 HWM 기준은 누적 수익 상태에서도 발동할 수 있음을 인지하고 승인할 것.
- 1-15. 단계별 조치:
  - -2%: 신규 비중 확대 크기 75%로 축소
  - -4%: 신규 비중 확대 크기 50%로 축소
  - -6%: 위험 노출 상한 25%로 축소, 현행 모델 동결
  - -8%: 신규 거래 전면 중단, 위험자산 축소 개시
  - -10%: 【yh확정】 전량 정리, 미체결 전부 취소, 현행 세대(Champion+Challenger 전부) 영구 폐기, 자본보존 상태 전환
- 1-16. -8%~-10% 구간의 갭 하락(장 시작 급락) 시나리오는 백테스트·스트레스 테스트에 의무 포함한다.
- 1-17. 재진입 게이트: -10% 발동 후에는 (a) 최소 1개월 자본보존 유지, (b) 새 세대가 4장 검증 파이프라인 전체를 처음부터 통과, (c) yh 수동 승인 1회 — 세 조건을 모두 만족해야 재진입한다.

### 무결성 차단
- 1-18. 데이터 누락·이상, 잔고 불일치(0원 초과), 거래소/API 장애 감지 시 신규 주문 전면 차단.
- 1-19. 모든 주문에 고유 ID 부여, 중복 주문 자동 차단. 주문 이벤트는 append-only 기록.
- 1-20. 시간은 내부 UTC 저장, 표시만 KST 변환.

## 2장. 자산 유니버스 (v1)

- 2-1. 국내상장 상품만 사용한다. 근거: 단일 장 마감(일일 파이프라인 단순화), 환전 비용 제거, 0-4 계좌 원칙과 일치.
- 2-2. v1 슬리브 (티커는 【S-3】 실사 후 확정):
  - 금: 상한 30% — 금현물형(언헤지) vs 골드선물(H)(환헤지) 중 택1 【D-3】【S-2】
  - 미국 달러: 상한 20% — 달러선물 ETF 또는 달러 RP/예수금 【S-3】
  - 엔: 상한 15% — 엔선물 ETF
  - 현금성: 최소 30% — 파킹형/초단기채
- 2-3. 은·유로는 v1에서 제외한다 【D-4】. 은은 변동성 프로파일이 상이하고, 유로는 국내상장 통화 수단이 마땅치 않음. v2 이후 재검토.
- 2-4. 환헤지 정책: 헤지/언헤지 상품을 같은 슬리브 안에서 혼용하지 않는다. 각 자산 정의에 헤지 여부를 명시한다.
- 2-5. 자산별 상한은 커널 소속(수정 불가), 자산별 목표 비중은 전략층 소속(학습으로 변경 가능).
- 2-6. 상관관계 급등 시(예: 금-달러 동조화) 해당 자산군 합산 위험 한도를 자동 축소한다.

## 3장. 운용 문법

- 3-1. 배분형 단일 문법을 사용한다. 개별 포지션 손절/익절 없음. 방어는 1장 낙폭 단계로 일원화한다.
- 3-2. 신호 계산 일 1회(일봉 기준). 정규 리밸런싱 주 1회. 위험 리밸런싱(1장 조치)은 즉시.
- 3-3. 목표 비중과 현재 비중 차이 2%p 미만이면 거래하지 않는다.
- 3-4. 자산당 비중 조정 하루 최대 1회, 전체 신규 거래 하루 최대 4건. 동일 자산 비중 재확대는 최소 2거래일 대기.
- 3-5. 거래 실행 조건(전부 충족): 예상 순수익 > 예상 총비용(수수료+스프레드+슬리피지+**세금**) × 3, 신호 신뢰도 ≥ 65%, 데이터 무결성 정상, 시장 국면 ≠ SHOCK/UNCERTAIN, 손실 한도 미도달.
- 3-6. 무거래 상태를 정상 판단으로 인정한다.
- 3-7. 통계용 라운드트립 정의: 특정 자산의 비중 증가 시작 시점부터 해당 증가분의 청산 완료 시점까지.
- 3-8. 비중 결정 기본: 역변동성 배분 × 신호 강도 × 국면 신뢰도, 이후 상관관계·집중도·자산 상한 반영.

## 4장. 학습·승격 거버넌스

- 4-1. 학습 데이터: 시장 전체 데이터를 주 교재로, 자기 운용 데이터(신호·거절 사유·실체결 비용·MFE/MAE)는 비용·슬리피지·체결 보정용으로 사용한다. 실행된 거래만으로 학습하지 않는다.
- 4-2. 프로덕션 Champion은 항상 1개. 신규 모델은 Challenger로 시작: TRAINED → BACKTESTED → VALIDATED(워크포워드, purge/embargo 적용) → SHADOW → CANARY → CHAMPION.
- 4-3. 챌린저 비교는 거래 건수 기반 지표(PF·승률)가 아니라 **섀도/카나리 포트폴리오의 일간 수익률 시계열**로 한다. 저빈도 운용에서 거래 표본은 통계적으로 무의미하다.
- 4-4. 검증 기간: 섀도 최소 60거래일, 카나리 최소 60거래일(각 약 1분기). 카나리 자본은 1-8에 따라 최대 10%.
- 4-5. 승격 조건(전부 충족): 비용 차감 기대수익 양수 / 일간 수익률 기준 위험조정 성과 Champion 이상 / 최대 낙폭 Champion 대비 미악화 / 비용 2배 스트레스에서 양수 유지 / 특정 자산·특정 기간 수익 의존도 각 40% 미만 / 데이터 누수 0건 / 섀도·카나리 운용 오류 0건 / 4-6 시나리오 라이브러리 통과.
- 4-6. 고정 스트레스 시나리오 라이브러리: 2013 금 폭락, 2020년 3월, 2022 강달러 급등 구간 등 지정 구간을 상수로 고정하고, 모든 챌린저는 해당 구간에서 파국적 실패(구간 낙폭이 커널 한도 초과) 없이 통과해야 한다. 구간 추가는 가능, 삭제는 금지.
- 4-7. 세대당 파라미터 변화 상한을 둔다(구체 수치는 SPEC-LEARNING에서 정의). 한 번의 승격으로 전략이 급변하는 것을 금지.
- 4-8. 학습 주기: 일일(성과 대조·드리프트 탐지·리포트) / 주간(보정·임계값 후보 생성, 실거래 미반영) / 월간(챌린저 재학습·검증 착수) / 분기(과최적화 검사·세대 평가).
- 4-9. 모든 승격·폐기 결정은 사유와 함께 기록하고 삭제하지 않는다.

## 5장. 통계·감사

- 5-1. 일일 통계 전량 수집 의무 【yh확정】. 상세 항목 목록은 SPEC-STATS로 분리(주문·성과·거래 품질·MFE/MAE·분해 통계·모델 통계·시스템 통계).
- 5-2. 모든 주문·신호·결정에 dataset/feature/model/strategy/risk-config/code-commit 버전을 연결한다. "특정일에 왜 샀는가"를 완전 재현할 수 있어야 한다.
- 5-3. 시장 원본 데이터 수정 금지, 백테스트 결과 덮어쓰기 금지, 과거 모델 파일 삭제 금지.
